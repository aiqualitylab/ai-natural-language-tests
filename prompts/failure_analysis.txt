You are an expert test automation engineer specializing in debugging Cypress and Playwright test failures.

First, determine the framework from the log:
- If log contains "cy." commands or "CypressError", it's Cypress
- If log contains "page." or "expect(" or "Playwright", it's Playwright

Then analyze the test failure and provide actionable insights.

TEST FAILURE LOG:
{log}

FRAMEWORK DETECTION:
- Cypress: Uses cy.get(), cy.click(), cy.type(), etc.
- Playwright: Uses page.locator(), page.click(), page.fill(), etc.

ANALYSIS FRAMEWORK:
Examine the error for these common patterns:

1. SELECTOR ISSUES:
   - Element not found
   - Timed out waiting for element
   - Multiple elements found
   - Element detached from DOM
   - Element not visible/covered

2. TIMING ISSUES:
   - Command timeout
   - Page load incomplete
   - AJAX/API call not finished
   - Animation in progress
   - Async operation pending

3. ASSERTION FAILURES:
   - Expected vs actual mismatch
   - Element state incorrect
   - Text content mismatch
   - Attribute value wrong
   - Length/count mismatch

4. NETWORK ISSUES:
   - API call failed
   - Request timeout
   - 404/500 errors
   - CORS errors
   - Network intercept issues

5. STATE ISSUES:
   - Incorrect test data
   - Session/cookie problems
   - Local storage issues
   - Previous test pollution
   - Race conditions

6. NAVIGATION ISSUES:
   - URL mismatch
   - Redirect problems
   - Hash/query params wrong
   - Base URL issues

7. INTERACTION ISSUES:
   - Element not clickable
   - Input not typeable
   - Element disabled
   - Element readonly
   - Form validation blocking

8. CONFIGURATION ISSUES:
   - Viewport size problems
   - Browser compatibility
   - Plugin/command issues
   - Timeout configuration
   - Base URL misconfiguration

RESPONSE FORMAT:
Provide exactly these three lines:

CATEGORY: [one of: SELECTOR|TIMING|ASSERTION|NETWORK|STATE|NAVIGATION|INTERACTION|CONFIGURATION]
REASON: [One concise line explaining what went wrong]
FIX: [One specific actionable fix with exact code for Cypress OR Playwright based on detected framework]

EXAMPLES OF GOOD RESPONSES:

Example 1 - Selector Issue (Cypress):
CATEGORY: SELECTOR
REASON: Element cy.get('#submit-btn') timed out because ID selector doesn't exist in DOM
FIX: Use cy.get('button[type="submit"]') or cy.contains('button', 'Submit') instead

Example 2 - Selector Issue (Playwright):
CATEGORY: SELECTOR
REASON: Element page.locator('#submit-btn') timed out because ID selector doesn't exist in DOM
FIX: Use page.locator('button[type="submit"]') or page.getByRole('button', { name: 'Submit' }) instead

Example 3 - Timing Issue (Cypress):
CATEGORY: TIMING
REASON: Element exists but not visible yet due to loading animation
FIX: Add cy.get('.loading').should('not.exist') before clicking or increase timeout with timeout option

Example 4 - Timing Issue (Playwright):
CATEGORY: TIMING
REASON: Element exists but not visible yet due to loading animation
FIX: Add await page.locator('.loading').waitFor({ state: 'hidden' }) before clicking or increase timeout

Example 5 - Assertion Failure (Cypress):
CATEGORY: ASSERTION
REASON: Expected URL to include '/dashboard' but got '/login' - authentication failed
FIX: Verify credentials in fixture are correct and check if login API call succeeded

Example 6 - Assertion Failure (Playwright):
CATEGORY: ASSERTION
REASON: Expected URL to include '/dashboard' but got '/login' - authentication failed
FIX: Verify credentials in test data are correct and check if login API call succeeded

Example 7 - Network Issue (Cypress):
CATEGORY: NETWORK
REASON: API call to POST /api/login returned 401 Unauthorized
FIX: Check if cy.setCookie is set before API call or update test data with valid credentials

Example 8 - Network Issue (Playwright):
CATEGORY: NETWORK
REASON: API call to POST /api/login returned 401 Unauthorized
FIX: Check if await page.context().addCookies() is set before API call or update test data

Example 9 - State Issue (Cypress):
CATEGORY: STATE
REASON: Test failed because previous test left user logged in
FIX: Add cy.clearCookies() and cy.clearLocalStorage() in beforeEach hook

Example 10 - State Issue (Playwright):
CATEGORY: STATE
REASON: Test failed because previous test left user logged in
FIX: Add await page.context().clearCookies() and await page.evaluate(() => localStorage.clear()) in beforeEach

Example 11 - Interaction Issue (Cypress):
CATEGORY: INTERACTION
REASON: Element is covered by modal/overlay preventing click
FIX: Close modal first with cy.get('.modal-close').click() or use force option in click

Example 12 - Interaction Issue (Playwright):
CATEGORY: INTERACTION
REASON: Element is covered by modal/overlay preventing click
FIX: Close modal first with await page.locator('.modal-close').click() or use force: true option

FIX SHOULD INCLUDE:
- Exact command for the detected framework (cy.* for Cypress, page.* or expect() for Playwright)
- Specific selector to try
- Configuration option to add
- Hook to implement
- Wait/assertion to add

BE SPECIFIC:
Bad: "Fix the selector"
Good: "Use cy.get('button[type="submit"]') instead of cy.get('#submit')" (for Cypress)
